# Evermist

[English](README.md) · **Русский**

Приложение для D&D: выводит карты на телевизор и добавляет **туман войны**, который можно стирать вживую со второго экрана.

![Снятие тумана в окне игрока](assets/reveal.gif)

Всё работает локально - одно окно остаётся у на ноутбуке, второе выводится на телевизор/проектор для игроков. Подземелье открывается постепенно, по мере исследования игроками.

> **ВАЖНО: это не полноценный VTT.** Здесь нет токенов, трекера инициативы и кубиков. Evermist умеет одно - показать карту и спрятать её часть. Зато делает это очень красиво.

## Возможности

- **Два экрана, синхронно.** Окно мастера и чистое окно игрока (без кнопок и курсора) для телевизора.
- **Живой туман войны.** Мягкий, с анимацией облаков, а не плоская чёрная заливка. Стирается кистью, или можно задать области показа и скрытия отдельными фигурами, которые потом можно менять.
- **Сетка.** Настраиваются тип (квадраты, гексы), размер, смещение, цвет и прозрачность.
- **Статичные и анимированные карты.** Поддерживаются JPG/PNG или анимированные карты в MP4/WebM (например, из Dungeon Alchemist).
- **Сцены.** Можно сохранять несколько карт и переключаться между ними прямо во время игры, с плавным переходом.
- **Большие карты.** Работает плавно даже на картах 10000×6000.

## Как это выглядит

**Анимированные карты** - видео-карты (вода, огонь факелов) проигрываются прямо под туманом.

**Инструменты-фигуры** - аккуратные комнаты вырезаются прямоугольником, кругом и многоугольником.

### Первый запуск

При первом запуске система покажет разовое предупреждение безопасности. Evermist бесплатный и **не подписан** цифровой подписью - сертификаты стоят денег.

- **Windows:** если появилось «Система Windows защитила ваш компьютер», нужно нажать «Подробнее», затем «Выполнить в любом случае».
- **macOS:** если появилось «Программу Evermist нельзя открыть, так как Apple не может её проверить…», кликнуть по приложению правой кнопкой мыши, выбрать «Открыть», а потом ещё раз «Открыть» в диалоге. (При обычном двойном клике этой кнопки в первый раз не будет.)
- **Linux:** нужно сделать AppImage исполняемым (`chmod +x Evermist-*.AppImage` или «Свойства → Права → Разрешить выполнение файла как программы») и запустить как обычно.

Система запомнит выбор, так будет только первый.

## С чего начать

Когда Evermist откроется, появится **окно мастера**. Это пульт управления.

1. **Открыть окно игрока.** В правой панели в разделе **Player** нажать **Open Window**. Появится второе окно, уже без кнопок - именно его видят игроки. Нужно перетащить его на телевизор или на второй монитор, и нажать **Fullscreen**.
2. **Загрузить карту.** Перетащить файл карты (JPG, PNG, MP4 или WebM) прямо в окно мастера. Файл загрузится как новая сцена и сразу будет полностью закрыт туманом.
3. **Открыть карту.** Выбрать инструмент на нижней панели и открывать то, что видит группа:
   - **Brush (кисть)** - стирает туман от руки (размер меняется клавишами `[` и `]`).
   - **Rectangle / Circle / Polygon** - рисуют ровную область, удобно для комнат и коридоров.
   - **Reveal / Shroud** - задаёт, открывает фигура область или прячет (например, чтобы закрыть дверь за спиной у группы).
   - **Select (V)** - перемещение, правка и удаление готовых областей.
4. **Настроить сетку.** В разделе **Grid** справа переключить тип сетки (квадраты или гексы) и подогнать размер и смещение под сетку на карте.
5. **Вывести на телевизор.** Когда включён **Auto** (раздел Player), все изменения сразу появляются на экране игрока. Для ручного режима - выключить Auto и жать **Send ▶** (или `Space`), когда пора открыть следующую комнату. Кнопка **Sync View** подгоняет камеру игрока под камеру мастера.
6. **Менять карты по ходу игры.** В панели **Scenes** можно сохранить несколько карт и переключаться между ними плавным затемнением. Удобно, когда партия переходит из таверны в подземелье и не хочется ломать атмосферу.

> **Совет:** клавиша `?` в окне мастера показывает полный список горячих клавиш.

## Запуск из исходников

Шага сборки нет - это обычный JavaScript в оболочке Electron.

```bash
npm install     # один раз, после клонирования
npm start       # запустить приложение
```

Собрать установщик под текущую платформу:

```bash
npm run build         # Windows portable .exe
npm run build:mac     # macOS .dmg
npm run build:linux   # Linux AppImage
```

Сборки под все три платформы GitHub Actions делает сам при пуше тега `v*` (см. файл [`.github/workflows/release.yml`](.github/workflows/release.yml)).

## О файлах карт

Сами карты (`.webm` / `.mp4`) и папка с данными приложения в репозиторий не входят, они лежат на диске рядом с программой. После клонирования остаётся только код. Карты добавляются уже через приложение.
