# Evermist

[English](README.md) · **Русский**

Приложение для D&D: выводит карты на телевизор и добавляет **туман войны**, который можно стирать вживую со второго экрана.

Всё работает локально - одно окно остаётся на ноутбуке, второе выводится на телевизор/проектор для игроков. Подземелье открывается постепенно, по мере исследования игроками.

> **Это не полноценный VTT.** Здесь нет токенов, трекера инициативы и кубиков. Evermist умеет одно - показать карту и спрятать её часть. Зато делает это очень красиво.

## Что умеет Evermist

### Два синхронных экрана

Есть окно мастера со всем управлением, и чистое окно игрока - без кнопок и курсора, только карта. Перетащите окно игрока на телевизор или проектор, включите полный экран, и игроки увидят только то, что вы захотите.

Откройте комнату на ноутбуке, и она сразу появится на телевизоре. Или переключитесь в ручной режим (Manual), подготовьте следующую комнату и раскройте её одной кнопкой, когда группа войдёт в дверь.

### Живой туман войны

Туман мягкий, с медленно плывущими облаками, а не плоская чёрная заливка. Открывайте карту, как удобно в моменте:

- **Brush (кисть)** - стирает туман от руки по мере движения группы.
- **Rectangle / Circle / Polygon** - вырезает ровные комнаты и коридоры.
- **Reveal / Shroud** - переключает режимы раскрытия и скрытия. Например, можно закрыть дверь за спиной у группы или вернуть их в темноту.
- **Select** - все фигуры остаются редактируемыми, так что область можно потом передвинуть или удалить.

![Рисование областей показа и скрытия](assets/tools.gif)

### Анимированные карты

Закиньте анимированную карту (MP4 или WebM, например экспорт из Dungeon Alchemist) - вода переливается, факелы мерцают, и всё это играет вживую под туманом.

![Анимированная карта под туманом](assets/animated-map.gif)

### Сетка под вашу карту

Переключайтесь между квадратами и гексами, затем подгоняйте размер, смещение, цвет и прозрачность, пока сетка не ляжет на сетку самой карты.

### Смена сцен по ходу игры

Сохраняйте несколько карт и переключайтесь между ними на лету, плавным затемнением. Партия выходит из таверны, экран растворяется, проявляется подземелье, и атмосфера не рвётся.

### Сделано для больших карт

Даже карта 10000×6000 двигается и масштабируется плавно. Загружайте детальные карты и не волнуйтесь.

> **Совет:** клавиша `?` в окне мастера показывает полный список горячих клавиш.

## Скачать

Свежую версию для вашей системы берите в [**Releases**](../../releases/latest):

| Система | Файл | Примечания |
|--------|------|-------|
| Windows | `Evermist-<версия>.exe` | Портативный, ставить не нужно, просто запускается |
| macOS | `Evermist-<версия>.dmg` | Универсальный (Intel и Apple Silicon) |
| Linux | `Evermist-<версия>.AppImage` | Сделать файл исполняемым и запустить |

Evermist бесплатный и не подписан цифровой подписью (сертификаты стоят денег), поэтому при первом запуске система покажет разовое предупреждение безопасности. Оно безобидное, вот как его пройти.

<details>
<summary>Как пройти предупреждение при первом запуске</summary>

- **Windows:** если появилось «Система Windows защитила ваш компьютер», нужно нажать «Подробнее», затем «Выполнить в любом случае».
- **macOS:** если появилось «Программу Evermist нельзя открыть, так как Apple не может её проверить…», кликнуть по приложению правой кнопкой мыши, выбрать «Открыть», а потом ещё раз «Открыть» в диалоге. (При обычном двойном клике этой кнопки в первый раз не будет.)
- **Linux:** нужно сделать AppImage исполняемым (`chmod +x Evermist-*.AppImage` или «Свойства → Права → Разрешить выполнение файла как программы») и запустить как обычно.

Система запомнит выбор, так что это только в первый раз.
</details>

## Запуск из исходников

Шага сборки нет - это обычный JavaScript в оболочке Electron.

```bash
npm install     # один раз, после клонирования
npm start       # запустить приложение
```

Собрать установщик под текущую платформу:

```bash
npm run build         # Windows portable .exe
npm run build:mac     # macOS .dmg
npm run build:linux   # Linux AppImage
```

Сборки под все три платформы GitHub Actions делает сам при пуше тега `v*` (см. файл [`.github/workflows/release.yml`](.github/workflows/release.yml)).

## О файлах карт

Сами карты (`.webm` / `.mp4`) и папка с данными приложения в репозиторий не входят, они лежат на диске рядом с программой. После клонирования остаётся только код. Карты добавляются уже через приложение.
